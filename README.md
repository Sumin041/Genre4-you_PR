# Genre4-you_PR
사용자 선호 기반 장르별 OTT 추천(룰 기반) 기획 문서.  
목표: **코드 구현 X**, Git/GitHub 협업 연습 O

## 진행 방식(브랜치 전략 요약)
- main은 항상 제출 가능 상태
- 작업은 작은 단위 브랜치(ex. feat/thriller-rules)에서 → PR → 리뷰 1인 → squash merge

## 폴더 안내
- docs/framework.md: 공통 작성 양식
- docs/<genre>/: 장르별 문서 (intro/rules/examples/notes)
- docs/summary.md: 장르 핵심 비교
- presentation/: 최종 발표 자료

## 역할/일정(예시)
- 스릴러/애니/로맨스/액션 각 1명 담당 → 주간 1회 PR
